Vibtrix API Documentation

Complete REST API reference for building the Vibtrix Flutter mobile application. This documentation covers all endpoints, authentication, request/response formats, and error handling.

API Version 1.0 ‚Ä¢ Last Updated: January 2026

üîê Authentication

JWT-based authentication with access and refresh tokens. Access tokens expire in 1 hour, refresh tokens in 7 days.

üì° Base URL

All API requests should be made to your server's base URL with the /api prefix.

üìÑ Response Format

All responses are in JSON format. Successful responses include data, errors include error messages.

‚ö° Rate Limiting

API requests are rate-limited. Auth endpoints: 5 req/min. General endpoints: 100 req/min.

Base URL

All API endpoints are relative to the base URL:

Production: https://your-domain.com/api
Development: http://localhost:3000/api
üí° Note for Flutter Developers Store the base URL in a configuration file or environment variable for easy switching between development and production environments.

Authentication

Vibtrix uses JWT (JSON Web Token) for authentication. The system provides two types of tokens:

Token Type Expiry Purpose
accessToken 1 hour Used for API authentication in the Authorization header
refreshToken 7 days Used to obtain new access tokens without re-authentication

Using Access Tokens

Include the access token in the Authorization header for all authenticated requests:

Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
‚ö†Ô∏è Token Storage Store tokens securely using Flutter's flutter_secure_storage package. Never store tokens in plain SharedPreferences.

Error Handling

All API errors follow a consistent format:

{
  "error": "Error message describing what went wrong",
  "details": [  // Optional: validation errors
    {
      "field": "email",
      "message": "Invalid email address"
    }
  ]
}

Common HTTP Status Codes

Code Meaning Action
200 Success Request completed successfully
400 Bad Request Check request body/parameters
401 Unauthorized Refresh token or re-authenticate
403 Forbidden User lacks permission for this action
404 Not Found Resource doesn't exist
409 Conflict Resource already exists (e.g., username taken)
429 Too Many Requests Rate limited - wait before retrying
500 Server Error Internal server error - retry later

Sign Up

POST /api/auth/signup

Register a new user account and receive authentication tokens.

Request Body

Field Type Required Description
username string Yes 3-20 characters, alphanumeric and underscores only
email string Yes Valid email address
password string Yes Minimum 8 characters

Example Request

{
  "username": "john_doe",
  "email": "john@example.com",
  "password": "securePassword123"
}

Success Response (200 OK)

{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": "cm1abc123def456",
    "username": "john_doe",
    "email": "john@example.com",
    "displayName": "john_doe",
    "avatarUrl": null,
    "role": "USER"
  }
}

Error Responses

  • 400 Validation failed - check request body format
  • 409 Username or email already taken
  • 429 Rate limited - too many registration attempts

Login (Generate Tokens)

POST /api/auth/token

Authenticate user and receive access and refresh tokens.

Request Body

Field Type Required Description
username string Yes Username or email address
password string Yes User's password

Example Request

{
  "username": "john_doe",
  "password": "securePassword123"
}

Success Response (200 OK)

{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": "cm1abc123def456",
    "username": "john_doe",
    "displayName": "John Doe",
    "avatarUrl": "https://utfs.io/f/avatar.jpg",
    "role": "USER"
  }
}

Error Responses

  • 400 Missing username or password
  • 401 Invalid credentials
  • 403 Account suspended or doesn't have permission to log in

Refresh Access Token

POST /api/auth/refresh

Generate a new access token using a valid refresh token.

Request Body

{
  "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}

Success Response (200 OK)

{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}

Error Responses

  • 400 Missing refresh token
  • 401 Invalid or expired refresh token
üí° Flutter Implementation Tip Implement an HTTP interceptor that automatically refreshes the access token when a 401 response is received. Store the new access token and retry the original request.

Forgot Password

POST /api/auth/forgot-password

Request a password reset OTP. For security, always returns success even if email not found.

Request Body

{
  "email": "john@example.com"
}

Success Response (200 OK)

{
  "message": "If an account with that email exists, a password reset OTP has been sent.",
  "success": true
}

Verify Reset OTP

POST /api/auth/verify-reset-otp

Verify the OTP sent for password reset.

Request Body

{
  "email": "john@example.com",
  "otp": "123456"
}

Success Response (200 OK)

{
  "success": true,
  "resetToken": "temporary-reset-token-uuid"
}
